# Taiwan Big Two AI (台灣大老二 AI 版)

本專案打造了一個現代化的大老二遊戲，基於 Electron 桌面應用框架。它結合了高效的啟發式 AI 與進階的多人格大型語言模型 (LLM) 研究引擎，專為自動化戰略對弈與學習分析而設計。

## 核心特色

### 1. 多人格研究架構 (Multi-Persona Architecture)
本專案支援多個獨立的 AI 人格，每個角色具備專屬的記憶與戰略偏好：
- **Diana (調適型學習)**：專注於平衡的對弈風格與戰術進化。
- **Ares (戰神)**：高侵略性人格，優先考慮戰力壓制與點數打擊。
- **BaseLLMAI 引擎**：統一的類別架構，處理算牌、盤面評估與敗局後的「自我反思 (Reflection)」。

### 2. 全自動化測試與學習
- **AFK 模式**：遊戲可進入全自動循環，讓 AI 代理人 24/7 持續對弈。
- **記憶進化**：AI 會在敗北後提煉結構化的「學習筆記」，儲存至持久化 JSON 記憶庫。
- **關鍵字匹配**：精煉的關鍵字相似度引擎，確保戰略規則能在不重複且不碎片化的情況下持續累積。

### 3. 嚴謹的台灣版規則
核心邏輯嚴格遵循傳統的**台灣大老二**玩法：
- **不含同花 (No Flush)**：五張同色牌但非順子者為無效牌型（與港版規則不同）。
- **牌型等級**：同花順 > 鐵支 > 葫蘆 > 順子。
- **花色強度**：黑桃 ♠ > 紅心 ♥ > 方塊 ♦ > 梅花 ♣。
- **一條龍**：13 張連號 (3-2) 即刻獲勝。

### 4. 開發者與研究工具
- **整合式測試**：執行 `npm test` 即可運行完整的測試套件 (Logic & UI)。
- **Microsoft Store 上架支援 (MSIX)**：完整支援符合商店規範的 4 位數版本號格式 (A.B.C.0)。
- **自動化建置流水線**：
  - **版本自動累加**：每次打包時會自動遞增版本編號（例如 `1.3.0.0`）。
  - **自動化檔名更新**：產生的安裝檔名會自動包含正確的 4 位數版本號。
  - **圖示工廠**：內建 PowerShell 腳本 (`scripts/generate-icons.ps1`)，可從單一原始 Logo 自動產生 Store 所需的所有 6 種特定尺寸圖示。
- **ESC 與外部點擊關閉**：所有彈窗與「關於」視窗均支援 `ESC` 鍵與「點擊外部區域」自動關閉，提升操作流暢度。

## 建置與安裝 (Installation & Build)

### 前置準備
- [Node.js](https://nodejs.org/) (建議 v16 或更高版本)
- [Git](https://git-scm.com/)

### 設定步驟
1. 複製專案庫：
   ```powershell
   git clone https://github.com/mesmerli/taiwan-big-two-ai.git
   ```
2. 安裝依賴套件：
   ```powershell
   npm install
   ```

### 執行測試
1. **邏輯測試**：`npm test`
2. **UI 測試**：`npm run test:ui`

### 本地執行
在開發模式下啟動應用程式：
```bash
npm start
```

### 封裝發布
1. **標準安裝檔 (EXE)**：`npm run dist`
2. **Microsoft Store 專用檔 (MSIX)**：`npm run dist:msix`

*註：執行這些指令會自動觸發版本累加並更新輸出檔案名稱。*

## 畫面操作說明 (UI Operation Guide)

### 遊戲操作
- **新遊戲**：點擊右上角的 **New Game** 按鈕即可重新開始。
- **選取手牌**：直接點擊手中的牌即可選取或取消選取。
- **出牌 / 過牌**：使用位於手牌上方的圓潤主按鈕。
- **強制「喊拉！」規則**：
  - 當您的出牌動作會導致手牌只剩 **1 張**時，您必須點擊**「喊拉！」**按鈕才能繼續。
  - 若您已經處於「喊拉」狀態，後續操作將受限於必須一次出完剩餘手牌或過牌。

### 角色與 AI 管理
- **切換角色**：點擊任何玩家的**頭像 (Avatar)** 即可在不同的人格（Alex, Bella, Chris, Diana, Ares）之間切換。
- **AI 設定**：點擊 LLM 角色旁的**齒輪圖示 (⚙️)** 可開啟設定視窗：
  - 設定 **API URL** 與 **模型 ID** (系統會自動偵測已載入的模型)。
  - **即時回饋**：若 API 無法連線，標籤旁會顯示鮮紅色的 **(連線失敗)**。
  - **記憶匯出/匯入**：將 AI 學習到的戰略規則存成 JSON 檔案。

### 系統功能
- **語系切換**：點擊右上角的 **全球圖標 (🌐)** 可切換語系。
- **規則與關於**：點擊 **藝術斜體 "i" 圖標** 進入規則說明或關於視窗。
- **關閉視窗**：點擊彈窗右上角的 **"X" 叉叉按鈕**，或直接按下 `ESC` 鍵。

## 搭配 LM Studio 使用 LLM

本專案的「深度學習」AI 角色（Diana 與 Ares）需要相容於 OpenAI 的 API 介面。建議使用 [LM Studio](https://lmstudio.ai/) 在本地端運行模型。

### 設定步驟
1. **下載 LM Studio**：前往 [lmstudio.ai](https://lmstudio.ai/)。
2. **下載模型**：搜尋並下載 `google/gemma-4-e2b`。
3. **啟動本地伺服器**：切換至 **Local Server** 分頁並點擊 **Start Server**。
4. **連接至遊戲**：開啟遊戲中的 **AI 設定 (⚙️)**，貼上 API URL 後即可自動偵測。
